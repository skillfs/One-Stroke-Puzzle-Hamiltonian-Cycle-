🐛 Bug #1 — "Dead End" นับผิดใน Mode 1

อาการ: Dead End = 23 และ Branching Avg = 0.00 ทุกครั้งไม่ว่า seed ไหน

สาเหตุ: นิยามเดิม = node ที่มี "free neighbor" เหลือ 0 — แต่ full Hamiltonian path ครอบทุก cell แล้ว ไม่มี free cell เหลือเลย ทุก node จึงเข้าเงื่อนไขเสมอ

วิธีแก้: เปลี่ยนเป็น 2 metric ใหม่ที่มีความหมายจริง:

  • Crossings — คู่ cell ติดกันบน grid แต่ไม่ต่อเนื่องใน path sequence
  • Decision Points — node ที่มี grid-neighbor ≥3 ตัวอยู่ใน path (solver อาจเลือกผิดทาง)

ยืนยันผล: 5×5 seed สุ่ม → Crossings=16, Decision Points=21 (ค่ามีความหมาย ไม่ตายตัว)

🐛 Bug #2 — Chain Mode วางบล็อกไม่ครบ

อาการ: ขอ 4 blocks ได้แค่ 2/4 หรือ 1/4 บ่อยครั้ง

สาเหตุ: ลองต่อบล็อกใหม่ด้วยแค่ 2 ทิศทาง (ปกติ + reverse) — ตัวเลือกไม่พอจะหาตำแหน่งที่ "head" ไปติดกับ "tail" เดิมได้

วิธีแก้: (1) เพิ่ม symmetries() สร้าง 8 ท่า (rotate×4 + flip×2 + reverse traversal) (2) เพิ่ม retry ทั้ง chain สูงสุด 40 รอบ เก็บ layout ที่ดีที่สุด

ก่อนแก้: 2/4, 2/4, 3/4, 1/4 (สุ่ม 6 seeds) → หลังแก้: 4/4 ทุก seed (8 seeds ทดสอบ)

🐛 Bug #3 — MST Wall-Following พังตอนมี Branching

อาการ: ทดสอบ tree รูป "กากบาท" (center ต่อ 4 แขน, degree=4) คาดหวัง path 20 cells ได้แค่ 4

สาเหตุ: right-hand-rule แบบง่ายที่ใช้ใส่ direction เดียวระดับ global ใช้ได้กับ "corridor กว้าง 1 ช่อง" เท่านั้น (chain/turn) พอเจอจุดแตกแขนง (degree≥3) มันเดินรอบแค่ block ตั้งต้นแล้วปิดวงกลับโดยไม่เข้า branch อื่นเลย

ทดสอบ: center+4 arms → expected 20, ได้ 4 (FAIL) ทดสอบ: chain 3 cells ตรง → expected 12, ได้ 12 (PASS) ทดสอบ: L-shape (มีเลี้ยว) → expected 16, ได้ 16 (PASS)

วิธีแก้ (เลือกใช้): จำกัด coarse-level ให้เป็น simple path (self-avoiding walk, degree≤2 เสมอ) แทน general spanning tree — หลีกเลี่ยง branching ไปเลย

วิธีแก้แบบสมบูรณ์ (ยังไม่ทำ): ต้องสร้าง per-node "port routing table" กำหนดว่า fine-cell ไหนในแต่ละ 2×2 block เชื่อมกับประตูไหน ตามจำนวน/ทิศทางของ tree edge ที่เข้า node นั้น

🐛 Bug #4 — Simple Path บางแบบยังพังอยู่ดี

อาการ: หลังแก้ #3 แล้ว ยังพังประปราย (4/15 seeds ที่ path=4 แทน 44)

สาเหตุ: ระหว่าง coarse cell 2 อันที่ติดกัน มี "ประตู" ที่ข้ามได้ 2 แถวคู่กันพร้อมกัน (เพราะแต่ละ coarse cell กว้าง 2 fine-cell) แต่ logic เปิดทั้ง 2 คู่เมื่อมี tree edge — ที่ถูกต้องควรเปิดแค่ 1 คู่ ทำให้เส้นมีทาง "ลัดย้อนกลับ" ก่อนสำรวจ block อื่นจนครบ

Debug: เปิด tree จริงที่ fail → trace path ทีละ step พบ: เข้า cell ข้างเคียงได้ 1 step แล้วย้อนกลับทันที ผ่าน "ประตู" คนละแถวกับที่เข้ามา (ควรเป็นประตูเดียวกัน)

วิธีแก้ (เลือกใช้): เพิ่ม retry-until-valid — ตรวจ path.length === visited.size×4 ถ้าไม่ตรง สุ่ม coarse path ใหม่ สูงสุด 30 รอบ (concept เดียวกับ Chain mode)

ผลหลังแก้: 15/15 และ 10/10 ผ่าน, retry เฉลี่ย 0-1 ครั้ง (ถูกมาก ไม่กระทบ performance)

📐 ข้อค้นพบเชิงทฤษฎีที่สำคัญ

  • Degree-1 Constraint — ใน Hamiltonian Path, node ที่มีเพื่อนบ้านเดียว (ทางตันแท้) ต้องเป็นจุดปลายเท่านั้น เพราะเส้นเดียวมีปลายได้แค่ 2 จุด → ใส่ "หนามยื่น" เกิน 2 จุดทำให้ด่านแก้ไม่ได้ทันที
  • Grid ≡ Graph — Grid ที่แต่ละ cell มี 4 neighbor คือ Grid Graph ชนิดหนึ่ง การเปลี่ยนมุมมองเป็น node/edge ไม่ลดความซับซ้อน เพราะข้อจำกัดมาจาก topology ของเส้นเดียว ไม่ใช่จาก representation
  • Longest Path = NP-hard — ไม่มี algorithm เร็วที่หา exact longest path บน grid ใหญ่ ต้องเลี่ยงด้วย heuristic (random walk) แทน exact search
  • Parity / Checkerboard Balance — ถ้าเจาะรูก่อนสร้าง path ต้องเช็ค (1) connectivity และ (2) checkerboard parity ต่างกันไม่เกิน 1 มิฉะนั้นไม่มี Hamiltonian Path
  • Spanning Tree → Hamiltonian Cycle (Wall-Following) — เทคนิคคลาสสิกจาก Snake-AI: ยุบ gridครึ่ง → spanning tree → เดินเลียบรอบด้วย right-hand rule → ได้ Hamiltonian cycle เสมอ (แต่ใช้ได้เต็มรูปแบบเฉพาะกรณีไม่แตกแขนง ตามที่ confirm จาก Bug #3)
  • Articulation Points = นิยาม Trap ที่ถูกต้อง — "ทางตันลวง" ที่ไม่ทำลาย Hamiltonian property ต้องมาจากจุดคอขวด (ตัดแล้วแยกด่านเป็น 2 ส่วน) ไม่ใช่จากการแตกกิ่งออกจาก path จริง (sprout จะกลายเป็น Tree ไม่ใช่ Path)

📚 เอกสารอ้างอิงที่ใช้

Oberdorf, R., Ferguson, A., Jacobsen, J. L., Kondev, J. (2006). Secondary structures in long compact polymers. Phys. Rev. E 74, 051801. — ที่มาของ Backbite move
Clisby, N. Hamiltonian Path Generator. clisby.net/projects/hamiltonian_path/ — อ้างอิง 2-phase generation + iteration formula
hand-burger. Hamiltonian Snake. github.com/hand-burger/hamiltonian-snake — เทียบ implementation จริงกับ pseudocode
คำแนะนำจากภายนอก (ผู้ใช้นำมา) — แนวคิด MST-Guided Random Walk (spanning tree + wall-following + sprout-based trap) นำมา red-team และปรับปรุง 2 จุดก่อนใช้จริง

ทฤษฎีกราฟพื้นฐาน (degree constraint, NP-hardness ของ Longest Path, Articulation Point/Tarjan's algorithm, Parity argument) เป็นความรู้ทั่วไปทาง Graph Theory ที่นำมาประยุกต์ผ่านการอภิปรายและวิเคราะห์ ไม่ได้อ้างจาก paper เฉพาะเจาะจง — ควรอ้างอิงตำรา Graph Theory มาตรฐานเพิ่มในเล่มจริง

🚫 เจาะรูก่อนสร้าง Path

แนวคิด: สุ่มเลือก cell เป็น "ช่องว่าง" ก่อน แล้วค่อยหา Hamiltonian path บนส่วนที่เหลือ

เหตุผลที่ตัดทิ้ง: ไม่การันตีว่ามี solution — ต้อง validate connectivity + checkerboard parity ก่อนทุกครั้ง ถ้าไม่ผ่านต้องสุ่มใหม่ (retry loop ที่ไม่รู้จะจบเมื่อไหร่ในกรณีเลวร้าย)

🚫 หา Longest Path บนพื้นที่ที่เจาะรูแล้ว

แนวคิด: เจาะ space ตามจำนวนที่ต้องการ แล้วให้ algorithm หาเส้นทางที่ยาวที่สุดในส่วนที่เหลือ

เหตุผลที่ตัดทิ้ง: Longest Path Problem เป็น NP-hard — DFS/backtracking ระเบิดเวลาบน grid ใหญ่ (8×8 ขึ้นไปช้ามาก) ไม่มี exact algorithm ที่เร็วพอ

🚫 Star Topology สำหรับ Chain Mode

แนวคิด: เชื่อม 3 sub-grid เข้าด้วยกันที่จุดศูนย์กลางจุดเดียว (เหมือนดาว)

เหตุผลที่ตัดทิ้ง: bridge cell ตรงกลางต้องถูกผ่าน 3 ครั้ง (เข้า 1 + ออก 1 + อีกแขนที่เหลือ) ขัดกฎ one-stroke ที่ผ่านแต่ละ cell ได้ครั้งเดียว → เปลี่ยนเป็น Chain (ต่อเรียงเป็นแถว) แทน

🚫 Sprout/แตกหน่อสร้าง Trap

แนวคิด: ระหว่างลาก path หลัก ให้สุ่ม "แตกกิ่งสั้นๆ" แทรกเข้าไปในพื้นที่ว่างข้างๆ เพื่อสร้างทางตันหลอกให้ผู้เล่นเดินผิด

เหตุผลที่ตัดทิ้ง: กิ่งที่แตกออกทำให้จุดแตกกิ่งมี degree 3 (เข้า+ออก+กิ่ง) และปลายกิ่งมี degree 1 — รวมกันเกิน 2 จุดที่ Hamiltonian Path อนุญาต → กลายเป็น Tree ไม่ใช่ Path → ด่านแก้ไม่ได้ทันที ใช้ Articulation Point (คอขวด) แทน ซึ่งไม่ทำลายความเป็น Path

🚫 ใช้ Node+Edge แทน Grid โดยตรง

แนวคิด: เปลี่ยน representation จาก grid 2D array เป็น graph (node+edge อิสระ) เพื่อหวังว่าจะลดความซับซ้อนหรือปลดล็อกอะไรบางอย่าง

เหตุผลที่ตัดทิ้ง: Grid ที่แต่ละ cell มี 4 neighbor คือ Grid Graph ชนิดหนึ่งอยู่แล้ว สมมูลกัน 100% — ข้อจำกัดเรื่อง dead-end มาจาก topology ของ "เส้นเดียว" ไม่ใช่จาก representation จึงไม่ช่วยอะไร (ดู diagram ในแท็บทฤษฎี)
